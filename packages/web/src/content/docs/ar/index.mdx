[**RAILWISE (甬算)**](/) هو وكيل ترميز بالذكاء الاصطناعي مفتوح المصدر. يتوفر كواجهة terminal، وتطبيق لسطح المكتب، أو إضافة لبيئة تطوير متكاملة (IDE).

![واجهة RAILWISE (甬算) في terminal مع سمة railwise](../../../assets/lander/screenshot.png)

لنبدأ.

---

#### المتطلبات

لاستخدام RAILWISE (甬算) في terminal، ستحتاج إلى:

1. محاكي terminal حديث مثل:
   - [WezTerm](https://wezterm.org) متعدد المنصات
   - [Alacritty](https://alacritty.org) متعدد المنصات
   - [Ghostty](https://ghostty.org) على Linux وmacOS
   - [Kitty](https://sw.kovidgoyal.net/kitty/) على Linux وmacOS

2. مفاتيح API لمزوّدي نماذج اللغة (LLM) الذين تريد استخدامهم.

## التثبيت

أسهل طريقة لتثبيت RAILWISE (甬算) هي عبر سكربت التثبيت.

```bash
curl -fsSL https://railwise.ai/install | bash
```

يمكنك أيضا تثبيته عبر الأوامر التالية:

- **باستخدام Node.js**

        <Tabs>

      <TabItem label="npm">
      ```bash
      npm install -g railwise-ai
      ```

          </TabItem>

        <TabItem label="Bun">
        ```bash
        bun install -g railwise-ai
        ```

          </TabItem>

        <TabItem label="pnpm">
        ```bash
        pnpm install -g railwise-ai
        ```

          </TabItem>

        <TabItem label="Yarn">
        ```bash
        yarn global add railwise-ai
        ```

      </TabItem>

  </Tabs>

- **باستخدام Homebrew على macOS وLinux**

  ```bash
  brew install anomalyco/tap/railwise
  ```

  > نوصي باستخدام tap الخاص بـ RAILWISE (甬算) للحصول على أحدث الإصدارات. صيغة `brew install railwise` الرسمية تُدار بواسطة فريق Homebrew ويتم تحديثها بوتيرة أقل.

- **باستخدام Paru على Arch Linux**

  ```bash
  sudo pacman -S railwise           # Arch Linux (Stable)
  paru -S railwise-bin              # Arch Linux (Latest from AUR)
  ```

#### Windows

:::tip[موصى به: استخدم WSL]
لأفضل تجربة على Windows، نوصي باستخدام [Windows Subsystem for Linux (WSL)](/docs/windows-wsl). يوفر أداء أفضل وتوافقا كاملا مع ميزات RAILWISE (甬算).
:::

- **باستخدام Chocolatey**

  ```bash
  choco install railwise
  ```

- **باستخدام Scoop**

  ```bash
  scoop install railwise
  ```

- **باستخدام NPM**

  ```bash
  npm install -g railwise-ai
  ```

- **باستخدام Mise**


- **باستخدام Docker**

دعم تثبيت RAILWISE (甬算) على Windows باستخدام Bun قيد العمل حاليا.

يمكنك أيضا تنزيل الملف التنفيذي من صفحة [Releases](https://github.com/anomalyco/railwise/releases).

## الإعداد

مع RAILWISE (甬算) يمكنك استخدام أي مزود LLM عبر إعداد مفاتيح API الخاصة به.

إذا كنت جديدا على استخدام مزوّدي LLM، فنوصي باستخدام [RAILWISE (甬算) Zen](/docs/zen).
إنها قائمة منتقاة من النماذج تم اختبارها والتحقق منها بواسطة فريق RAILWISE (甬算).

1. شغّل الأمر `/connect` في واجهة TUI، واختر railwise، ثم انتقل إلى [railwise.ai/auth](https://railwise.ai/auth).

   ```txt
   /connect
   ```

2. سجّل الدخول، وأضف تفاصيل الدفع، ثم انسخ مفتاح API.

3. الصق مفتاح API.

   ```txt
   ┌ API key
   │
   │
   └ enter
   ```

بدلا من ذلك يمكنك اختيار أحد المزوّدين الآخرين. [اعرف المزيد](/docs/providers#directory).

## التهيئة

بعد أن قمت بإعداد مزوّد، يمكنك الانتقال إلى المشروع الذي تريد العمل عليه.

```bash
cd /path/to/project
```

ثم شغّل RAILWISE (甬算).

```bash
railwise
```

بعد ذلك، هيّئ RAILWISE (甬算) للمشروع عبر تشغيل الأمر التالي.

```bash frame="none"
/init
```

سيجعل هذا RAILWISE (甬算) يحلل مشروعك ويُنشئ ملف `AGENTS.md` في جذر المشروع.

:::tip
يستحسن أن تقوم بعمل commit لملف `AGENTS.md` الخاص بمشروعك إلى Git.
:::

يساعد هذا RAILWISE (甬算) على فهم بنية المشروع وأنماط الترميز المستخدمة.

## الاستخدام

أصبحت الآن جاهزا لاستخدام RAILWISE (甬算) للعمل على مشروعك. لا تتردد في سؤاله أي شيء!

إذا كنت جديدا على استخدام وكيل ترميز بالذكاء الاصطناعي، فإليك بعض الأمثلة التي قد تساعد.

---

### اطرح أسئلة

يمكنك أن تطلب من RAILWISE (甬算) شرح قاعدة الشيفرة لك.

:::tip
استخدم المفتاح `@` للبحث التقريبي عن الملفات داخل المشروع.
:::

```txt frame="none" "@packages/functions/src/api/index.ts"
How is authentication handled in @packages/functions/src/api/index.ts
```

يفيد هذا عندما تكون هناك أجزاء من قاعدة الشيفرة لم تعمل عليها.

---

### أضف ميزات

يمكنك أن تطلب من RAILWISE (甬算) إضافة ميزات جديدة إلى مشروعك، لكننا نوصي أولا بطلب إنشاء خطة.

1. **أنشئ خطة**

   لدى RAILWISE (甬算) _وضع Plan_ يعطل قدرته على إجراء التغييرات، ويقترح بدلا من ذلك _كيف_ سينفّذ الميزة.

   انتقل إليه باستخدام مفتاح **Tab**. سترى مؤشرا لذلك في الزاوية السفلية اليمنى.

   ```bash frame="none" title="التبديل إلى وضع Plan"
   <TAB>
   ```

   الآن لنصف ما نريده أن يفعله.

   ```txt frame="none"
   When a user deletes a note, we'd like to flag it as deleted in the database.
   Then create a screen that shows all the recently deleted notes.
   From this screen, the user can undelete a note or permanently delete it.
   ```

   احرص على تزويد RAILWISE (甬算) بتفاصيل كافية ليفهم ما تريد. يساعد أن تتحدث إليه كما لو كنت تتحدث إلى مطور مبتدئ ضمن فريقك.

   :::tip
   أعطِ RAILWISE (甬算) سياقا وأمثلة كافية لمساعدته على فهم ما تريد.
   :::

2. **حسّن الخطة**

   بعد أن يقدم لك خطة، يمكنك إعطاؤه ملاحظات أو إضافة تفاصيل أكثر.

   ```txt frame="none"
   We'd like to design this new screen using a design I've used before.
   [Image #1] Take a look at this image and use it as a reference.
   ```

   :::tip
   اسحب الصور وأفلِتها داخل terminal لإضافتها إلى الطلب.
   :::

   يستطيع RAILWISE (甬算) فحص أي صور تزوده بها وإضافتها إلى الطلب. يمكنك فعل ذلك عبر سحب الصورة وإفلاتها داخل terminal.

3. **ابنِ الميزة**

   عندما تشعر بالارتياح للخطة، عد إلى _وضع Build_ بالضغط على مفتاح **Tab** مرة أخرى.

   ```bash frame="none"
   <TAB>
   ```

   ثم اطلب منه تنفيذ التغييرات.

   ```bash frame="none"
   Sounds good! Go ahead and make the changes.
   ```

### نفّذ تغييرات

للتغييرات الأبسط، يمكنك أن تطلب من RAILWISE (甬算) تنفيذها مباشرة دون الحاجة لمراجعة خطة أولا.

```txt frame="none" "@packages/functions/src/settings.ts" "@packages/functions/src/notes.ts"
We need to add authentication to the /settings route. Take a look at how this is
handled in the /notes route in @packages/functions/src/notes.ts and implement
the same logic in @packages/functions/src/settings.ts
```

تأكد من تقديم قدر مناسب من التفاصيل لكي يجري RAILWISE (甬算) التغييرات الصحيحة.

---

### تراجع عن التغييرات

لنفترض أنك طلبت من RAILWISE (甬算) إجراء بعض التغييرات.

```txt frame="none" "@packages/functions/src/api/index.ts"
Can you refactor the function in @packages/functions/src/api/index.ts?
```

لكن تتضح لك لاحقا أنها ليست ما تريده. يمكنك **التراجع** عن التغييرات باستخدام الأمر `/undo`.

```bash frame="none"
/undo
```

سيقوم RAILWISE (甬算) الآن بعكس التغييرات التي أُجريت ويعرض رسالتك الأصلية مجددا.

```txt frame="none" "@packages/functions/src/api/index.ts"
Can you refactor the function in @packages/functions/src/api/index.ts?
```

من هنا يمكنك تعديل الطلب وطلب المحاولة مرة أخرى.

:::tip
يمكنك تشغيل `/undo` عدة مرات للتراجع عن عدة تغييرات.
:::

أو يمكنك **إعادة تنفيذ** التغييرات باستخدام الأمر `/redo`.

```bash frame="none"
/redo
```
