Selv om RAILWISE (甬算) kan kjøre direkte på Windows, anbefaler vi å bruke [Windows Subsystem for Linux (WSL)](https://learn.microsoft.com/en-us/windows/wsl/install) for best mulig opplevelse. WSL gir et Linux-miljø som fungerer godt med funksjonene i RAILWISE (甬算).

:::tip[Hvorfor WSL?]
WSL gir bedre filsystemytelse, full terminalstøtte og kompatibilitet med utviklingsverktøyene RAILWISE (甬算) er avhengig av.
:::

---

## Oppsett

<Steps>

1. **Installer WSL**

   Hvis du ikke har gjort det ennå, [installer WSL](https://learn.microsoft.com/en-us/windows/wsl/install) med den offisielle guiden fra Microsoft.

2. **Installer RAILWISE (甬算) i WSL**

   Når WSL er satt opp, åpner du WSL-terminalen og installerer RAILWISE (甬算) med en av [installasjonsmetodene](/docs/).

   ```bash
   curl -fsSL https://railwise.ai/install | bash
   ```

3. **Bruk RAILWISE (甬算) fra WSL**

   Gå til prosjektmappen din (Windows-filer er tilgjengelige via `/mnt/c/`, `/mnt/d/` osv.) og kjør RAILWISE (甬算).

   ```bash
   cd /mnt/c/Users/YourName/project
   railwise
   ```

</Steps>

## Skrivebordsapp + WSL-server

Hvis du foretrekker RAILWISE (甬算)-skrivebordsappen, men vil kjøre serveren i WSL:

1. **Start serveren i WSL** med `--hostname 0.0.0.0` for å tillate eksterne tilkoblinger:

   ```bash
   railwise serve --hostname 0.0.0.0 --port 4096
   ```

2. **Koble skrivebordsappen** til `http://localhost:4096`

:::note
Hvis `localhost` ikke fungerer i oppsettet ditt, bruk WSL-IP-adressen i stedet (fra WSL: `hostname -I`) og koble til `http://<wsl-ip>:4096`.
:::

:::caution
Når du bruker `--hostname 0.0.0.0`, sett `RAILWISE_SERVER_PASSWORD` for å sikre serveren.

```bash
RAILWISE_SERVER_PASSWORD=your-password railwise serve --hostname 0.0.0.0
```

:::

## Web-klient + WSL

For best web-opplevelse på Windows:

1. **Kjør `railwise web` i WSL-terminalen** i stedet for PowerShell:

   ```bash
   railwise web --hostname 0.0.0.0
   ```

2. **Åpne i Windows-nettleseren** på `http://localhost:<port>` (RAILWISE (甬算) skriver ut URL-en)

Når du kjører `railwise web` fra WSL, får du riktig filsystemtilgang og terminalintegrasjon, samtidig som det er tilgjengelig fra Windows-nettleseren din.

## Tilgang til Windows-filer

WSL kan få tilgang til alle Windows-filer via `/mnt/`-katalogen:

- `C:`-stasjon → `/mnt/c/`
- `D:`-stasjon → `/mnt/d/`
- Og så videre...

Eksempel:

```bash
cd /mnt/c/Users/YourName/Documents/project
railwise
```

:::tip
For en jevnere opplevelse kan du klone/kopiere repoet ditt inn i WSL-filsystemet (for eksempel under `~/code/`) og kjøre RAILWISE (甬算) der.
:::

## Tips

- Kjør RAILWISE (甬算) i WSL for prosjekter lagret på Windows-stasjoner: filtilgang fungerer smidig
- Bruk [WSL-utvidelsen i VS Code](https://code.visualstudio.com/docs/remote/wsl) sammen med RAILWISE (甬算) for en integrert utviklingsflyt
- RAILWISE (甬算)-konfigurasjon og sesjoner lagres i WSL-miljøet på `~/.local/share/railwise/`
