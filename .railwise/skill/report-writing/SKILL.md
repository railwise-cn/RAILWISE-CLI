## 适用场景

- 监测日报、周报、月报编制
- 年度变形监测总结报告
- 投标技术标书编写
- 工程勘察报告起草

## 核心行文原则

1. **客观第三方视角**：用"经监测"、"监测结果显示"，禁用"我们发现"
2. **数据驱动结论**：每个结论后必须有数据支撑，禁止空洞表述
3. **被动句式**：优先使用"已完成"而非"我们完成了"
4. **阿拉伯数字**：量值统一使用阿拉伯数字，不用汉字数字
5. **规范名称规范**：首次出现用全称，后续可用括号简称，如"《城市轨道交通工程监测技术规范》（GB 50911，下称'监测规范'）"

## 禁止使用的表达

| ❌ 禁用 | ✅ 替换 |
|--------|--------|
| 经过我们的分析 | 经分析 |
| 感觉数据正常 | 监测数据表明结构处于稳定状态 |
| 沉降量很小 | 本周期最大沉降量为0.3mm |
| 好像没问题 | 各监测指标均未超过预警值 |
| 建议加强监测 | 建议将监测频率提升至1次/天 |

## 监测日报标准结构

```markdown
# [项目名称] 监测日报

**报告编号**：RW-YYYY-XXXX-NNN
**监测日期**：YYYY年MM月DD日
**天气情况**：晴/阴/雨，气温XX～XX℃

---

## 1. 本日监测概况

本日对[工程名称]开展第三方监测，共完成[项目]等XX项监测工作，累计观测测点XX个。

## 2. 各监测项目成果汇总

### 2.1 [监测项目名称]

[数据表格]

### 2.2 [监测项目名称]（如有）

[数据表格]

## 3. 预警情况

> 本日无预警情况。 / ⚠️ 以下测点数据异常，请相关方关注：

[预警内容或"无"]

## 4. 当日施工工况记录

[简述当日施工内容，如"进行第三层土方开挖，开挖深度约XX.Xm"]

## 5. 结论与建议

经本日监测，[工程名称]各监测指标均在控制范围内，结构整体处于稳定状态。建议按正常频率继续监测。
```

## 总结报告核心章节

1. **前言**（依据文件、监测目的、报告范围）
2. **工程概况**（地理位置、工程规模、周边环境）
3. **监测内容与方案概述**（引用规范、测点布设说明）
4. **仪器设备及精度**（设备清单及检定情况）
5. **监测成果综合分析**（按监测项目分节，含数据表格和趋势描述）
6. **变形规律分析**（与工程工况的相关性分析）
7. **安全评价结论**（明确"本工程监测期间结构处于[稳定/基本稳定/异常]状态"）
8. **建议事项**（后续监测建议或遗留问题）
9. **附件**（监测点位布置图、特征点变化曲线图）

## 投标技术标书要素

| 章节 | 关键内容 | 评分要点 |
|-----|---------|---------|
| 技术方案理解 | 识别重难点，提出针对性措施 | 契合度、专业性 |
| 监测方案 | 测点布设图、仪器选型、频率设定 | 完整性、规范符合性 |
| 质量保证 | ISO体系、质检流程、样本报告 | 制度完备性 |
| 项目业绩 | 类似项目合同及完工证明 | 相关度、规模 |
| 人员配置 | 项目负责人执业资格、团队经验 | 资质等级 |
| 应急预案 | 超限应急响应流程、24小时值班 | 响应时效性 |

## 专业术语规范对照

| 正确用法 | 错误用法 |
|---------|---------|
| 累计变化量 | 总变形量 |
| 本期变化量 | 本次变化量（单词"次"不够专业） |
| 变化速率（mm/d） | 变化速度 |
| 控制值 / 报警值 | 限值（笼统，须区分预警值和报警值） |
| 监测基准值 | 初始值（须说明取均值还是单次） |
| 收敛 | 稳定（需满足特定判断标准才能用"稳定"） |
