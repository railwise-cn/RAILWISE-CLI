CLI railwise по умолчанию запускает [TUI](/docs/tui) при запуске без каких-либо аргументов.

```bash
railwise
```

Но он также принимает команды, описанные на этой странице. Это позволяет вам программно взаимодействовать с railwise.

```bash
railwise run "Explain how closures work in JavaScript"
```

---

### tui

Запустите TUI railwise.

```bash
railwise [project]
```

#### Флаги

| Флаг         | Короткий | Описание                                                                      |
| ------------ | -------- | ----------------------------------------------------------------------------- |
| `--continue` | `-c`     | Продолжить последний сеанс                                                    |
| `--session`  | `-s`     | Идентификатор сеанса для продолжения                                          |
| `--fork`     |          | Разветвить сеанс при продолжении (используйте с `--continue` или `--session`) |
| `--prompt`   |          | Промпт для использования                                                      |
| `--model`    | `-m`     | Модель для использования в виде поставщика/модели.                            |
| `--agent`    |          | Агент для использования                                                       |
| `--port`     |          | Порт для прослушивания                                                        |
| `--hostname` |          | Имя хоста для прослушивания                                                   |

## Команды

CLI railwise также имеет следующие команды.

---

### agent

Управляйте агентами для railwise.

```bash
railwise agent [command]
```

---

### attach

Подключите терминал к уже работающему внутреннему серверу railwise, запущенному с помощью команд `serve` или `web`.

```bash
railwise attach [url]
```

Это позволяет использовать TUI с удаленным сервером railwise. Например:

```bash
# Start the backend server for web/mobile access
railwise web --port 4096 --hostname 0.0.0.0

# In another terminal, attach the TUI to the running backend
railwise attach http://10.20.30.40:4096
```

#### Флаги

| Флаг        | Короткий | Описание                             |
| ----------- | -------- | ------------------------------------ |
| `--dir`     |          | Рабочий каталог для запуска TUI      |
| `--session` | `-s`     | Идентификатор сеанса для продолжения |

### auth

Команда для управления учетными данными и входом в систему для провайдеров.

```bash
railwise auth [command]
```

---

#### login

railwise использует список провайдеров с [Models.dev](https://models.dev), поэтому вы можете использовать `railwise auth login` для настройки ключей API для любого поставщика, которого вы хотите использовать. Это хранится в `~/.local/share/railwise/auth.json`.

```bash
railwise auth login
```

Когда railwise запускается, он загружает поставщиков из файла учетных данных. И если в ваших средах определены какие-либо ключи или файл `.env` в вашем проекте.

---

#### list

Перечисляет всех проверенных поставщиков, которые хранятся в файле учетных данных.

```bash
railwise auth list
```

Или короткая версия.

```bash
railwise auth ls
```

---

#### logout

Выключает вас из провайдера, удаляя его из файла учетных данных.

```bash
railwise auth logout
```

#### install

Установите агент GitHub в свой репозиторий.

```bash

```

Это настроит необходимый рабочий процесс GitHub Actions и проведет вас через процесс настройки. [Подробнее](/docs/github).

### mcp

Управляйте серверами протокола контекста модели.

```bash
railwise mcp [command]
```

---

#### add

Добавьте сервер MCP в свою конфигурацию.

```bash
railwise mcp add
```

Эта команда поможет вам добавить локальный или удаленный сервер MCP.

---

#### list

Перечислите все настроенные серверы MCP и состояние их подключения.

```bash
railwise mcp list
```

Или используйте короткую версию.

```bash
railwise mcp ls
```

---

#### auth

Аутентификация с помощью сервера MCP с поддержкой OAuth.

```bash
railwise mcp auth [name]
```

Если вы не укажете имя сервера, вам будет предложено выбрать один из доступных серверов с поддержкой OAuth.

Вы также можете перечислить серверы с поддержкой OAuth и их статус аутентификации.

```bash
railwise mcp auth list
```

Или используйте короткую версию.

```bash
railwise mcp auth ls
```

---

#### logout

Удалите учетные данные OAuth для сервера MCP.

```bash
railwise mcp logout [name]
```

---

#### debug

Отладка проблем с подключением OAuth для сервера MCP.

```bash
railwise mcp debug <name>
```

### models

Перечислите все доступные модели от настроенных поставщиков.

```bash
railwise models [provider]
```

Эта команда отображает все модели, доступные у настроенных вами поставщиков, в формате `provider/model`.

Это полезно для определения точного названия модели, которое будет использоваться в [вашем config](/docs/config/).

При желании вы можете передать идентификатор поставщика, чтобы фильтровать модели по этому поставщику.

```bash
railwise models anthropic
```

#### Флаги

| Флаг        | Описание                                                                          |
| ----------- | --------------------------------------------------------------------------------- |
| `--refresh` | Обновите кеш моделей на сайте models.dev.                                         |
| `--verbose` | Используйте более подробный вывод модели (включая метаданные, такие как затраты). |

Используйте флаг `--refresh` для обновления списка кэшированных моделей. Это полезно, когда к поставщику добавлены новые модели и вы хотите увидеть их в railwise.

```bash
railwise models --refresh
```

### run

Запустите railwise в неинтерактивном режиме, передав приглашение напрямую.

```bash
railwise run [message..]
```

Это полезно для создания сценариев, автоматизации или когда вам нужен быстрый ответ без запуска полного TUI. Например.

```bash "railwise run"
railwise run Explain the use of context in Go
```

Вы также можете подключиться к работающему экземпляру `railwise serve`, чтобы избежать холодной загрузки сервера MCP при каждом запуске:

```bash
# Start a headless server in one terminal
railwise serve

# In another terminal, run commands that attach to it
railwise run --attach http://localhost:4096 "Explain async/await in JavaScript"
```

#### Флаги

| Флаг         | Короткий | Описание                                                                         |
| ------------ | -------- | -------------------------------------------------------------------------------- |
| `--command`  |          | Команда для запуска, используйте сообщение для аргументов                        |
| `--continue` | `-c`     | Продолжить последний сеанс                                                       |
| `--session`  | `-s`     | Идентификатор сеанса для продолжения                                             |
| `--fork`     |          | Разветвить сеанс при продолжении (используйте с `--continue` или `--session`)    |
| `--share`    |          | Поделиться сеансом                                                               |
| `--model`    | `-m`     | Модель для использования в виде поставщика/модели.                               |
| `--agent`    |          | Агент для использования                                                          |
| `--file`     | `-f`     | Файл(ы) для прикрепления к сообщению                                             |
| `--format`   |          | Формат: по умолчанию (отформатированный) или json (необработанные события JSON). |
| `--title`    |          | Название сеанса (использует усеченное приглашение, если значение не указано)     |
| `--attach`   |          | Подключитесь к работающему серверу railwise (например, http://localhost:4096)    |
| `--port`     |          | Порт локального сервера (по умолчанию случайный порт)                            |

### serve

Запустите автономный сервер railwise для доступа к API. Полный HTTP-интерфейс можно найти в [server docs](/docs/server).

```bash
railwise serve
```

При этом запускается HTTP-сервер, который обеспечивает доступ API к функциям railwise без интерфейса TUI. Установите `RAILWISE_SERVER_PASSWORD`, чтобы включить базовую аутентификацию HTTP (имя пользователя по умолчанию — `railwise`).

#### Флаги

| Флаг         | Описание                                                      |
| ------------ | ------------------------------------------------------------- |
| `--port`     | Порт для прослушивания                                        |
| `--hostname` | Имя хоста для прослушивания                                   |
| `--mdns`     | Включить обнаружение mDNS                                     |
| `--cors`     | Дополнительные источники браузера, позволяющие разрешить CORS |

#### list

```bash
railwise session list
```

##### Флаги

| ------------- | -------- | ----------------------------------------- |
| `--max-count` | `-n`     | Ограничить N последних сеансов.           |

### stats

Покажите статистику использования токенов и затрат для ваших сеансов railwise.

```bash
railwise stats
```

#### Флаги

| Флаг        | Описание                                                                                                   |
| ----------- | ---------------------------------------------------------------------------------------------------------- |
| `--days`    | Показать статистику за последние N дней (все время)                                                        |
| `--tools`   | Количество инструментов для отображения (все)                                                              |
| `--models`  | Показать разбивку по использованию модели (по умолчанию скрыто). Передайте номер, чтобы показать верхнюю N |
| `--project` | Фильтровать по проекту (все проекты, пустая строка: текущий проект)                                        |

---

### export

Экспортируйте данные сеанса в формате JSON.

```bash
railwise export [sessionID]
```

Если вы не укажете идентификатор сеанса, вам будет предложено выбрать один из доступных сеансов.

---

### import

Импортируйте данные сеанса из файла JSON или URL-адреса общего ресурса railwise.

```bash
railwise import <file>
```

Вы можете импортировать из локального файла или URL-адреса общего ресурса railwise.

```bash
railwise import session.json
railwise import https://opncd.ai/s/abc123
```

| ------------ | ------------------------------------------------------------- |
| `--port`     | Порт для прослушивания                                        |
| `--hostname` | Имя хоста для прослушивания                                   |
| `--mdns`     | Включить обнаружение mDNS                                     |

### acp

Запустите сервер ACP (агент-клиентский протокол).

```bash
railwise acp
```

Эта команда запускает сервер ACP, который обменивается данными через stdin/stdout с использованием nd-JSON.

#### Флаги

| Флаг         | Описание                    |
| ------------ | --------------------------- |
| `--cwd`      | Рабочий каталог             |
| `--port`     | Порт для прослушивания      |
| `--hostname` | Имя хоста для прослушивания |

---

### uninstall

Удалите railwise и удалите все связанные файлы.

```bash
railwise uninstall
```

#### Флаги

| Флаг            | Короткий | Описание                                   |
| --------------- | -------- | ------------------------------------------ |
| `--keep-config` | `-c`     | Сохраняйте файлы конфигурации              |
| `--keep-data`   | `-d`     | Храните данные сеанса и снимки             |
| `--dry-run`     |          | Покажите, что было бы удалено без удаления |
| `--force`       | `-f`     | Пропустить запросы подтверждения           |

---

### upgrade

Обновляет railwise до последней версии или определенной версии.

```bash
railwise upgrade [target]
```

Чтобы обновиться до последней версии.

```bash
railwise upgrade
```

Для обновления до определенной версии.

```bash
railwise upgrade v0.1.48
```

#### Флаги

| Флаг       | Короткий | Описание                                                  |
| ---------- | -------- | --------------------------------------------------------- |
| `--method` | `-m`     | Используемый метод установки: local, npm, pnpm, bun, brew |

| `RAILWISE_ENABLE_EXPERIMENTAL_MODELS` | логическое значение | Включить экспериментальные модели                                                |
| `RAILWISE_DISABLE_AUTOCOMPACT`        | логическое значение | Отключить автоматическое сжатие контекста                                        |

| `RAILWISE_CLIENT`                     | строка              | Идентификатор клиента (по умолчанию `cli`)                                       |

| `RAILWISE_SERVER_PASSWORD`            | строка              | Включить базовую аутентификацию для `serve`/`web`                                |
| `RAILWISE_SERVER_USERNAME`            | строка              | Переопределить имя пользователя базовой аутентификации (по умолчанию `railwise`) |
