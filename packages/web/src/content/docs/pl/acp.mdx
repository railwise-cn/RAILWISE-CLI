RAILWISE (甬算) obsługuje [Agent Client Protocol](https://agentclientprotocol.com) (ACP), który umożliwia korzystanie z RAILWISE (甬算) bezpośrednio w edytorach i środowiskach IDE.

:::tip
Listę edytorów i narzędzi obsługujących ACP znajdziesz w [raporcie postępu ACP](https://zed.dev/blog/acp-progress-report#available-now).
:::

ACP to otwarty standard, który ujednolica komunikację między edytorami kodu a agentami kodującymi AI.

---

## Konfiguracja

Aby używać RAILWISE (甬算) przez ACP, skonfiguruj swój edytor tak, aby uruchamiał polecenie `railwise acp`.

Uruchamia to RAILWISE (甬算) jako podproces zgodny z ACP, który komunikuje się z edytorem za pomocą JSON-RPC przez stdio.

Poniżej znajdują się przykłady dla edytorów obsługujących ACP.

---

### Zed

Dodaj do konfiguracji [Zed](https://zed.dev) (`~/.config/zed/settings.json`):

```json title="~/.config/zed/settings.json"
{
  "agent_servers": {
    "RAILWISE (甬算)": {
      "command": "railwise",
      "args": ["acp"]
    }
  }
}
```

Aby użyć, wybierz `agent: new thread` w **Palecie poleceń** (Command Palette).

Możesz także przypisać skrót klawiszowy, edytując `keymap.json`:

```json title="keymap.json"
[
  {
    "bindings": {
      "cmd-alt-o": [
        "agent::NewExternalAgentThread",
        {
          "agent": {
            "custom": {
              "name": "RAILWISE (甬算)",
              "command": {
                "command": "railwise",
                "args": ["acp"]
              }
            }
          }
        }
      ]
    }
  }
]
```

---

### JetBrains IDE

Dodaj do swojego [JetBrains IDE](https://www.jetbrains.com/) plik `acp.json` zgodnie z [dokumentacją](https://www.jetbrains.com/help/ai-assistant/acp.html):

```json title="acp.json"
{
  "agent_servers": {
    "RAILWISE (甬算)": {
      "command": "/absolute/path/bin/railwise",
      "args": ["acp"]
    }
  }
}
```

Następnie wybierz „RAILWISE (甬算)” w selektorze agentów AI Chat.

---

### Avante.nvim

Dodaj do konfiguracji [Avante.nvim](https://github.com/yetone/avante.nvim):

```lua
{
  acp_providers = {
    ["railwise"] = {
      command = "railwise",
      args = { "acp" }
    }
  }
}
```

Jeśli potrzebujesz przekazać klucz API:

```lua {6-8}
{
  acp_providers = {
    ["railwise"] = {
      command = "railwise",
      args = { "acp" },
      env = {
        RAILWISE_API_KEY = os.getenv("RAILWISE_API_KEY")
      }
    }
  }
}
```

### CodeCompanion.nvim

Aby używać RAILWISE (甬算) jako agenta ACP w [CodeCompanion.nvim](https://github.com/olimorris/codecompanion.nvim), dodaj następującą konfigurację do Neovim:

```lua
require("codecompanion").setup({
  interactions = {
    chat = {
      adapter = {
        name = "railwise",
        model = "claude-sonnet-4",
      },
    },
  },
})
```

Ta konfiguracja sprawia, że CodeCompanion używa RAILWISE (甬算) jako agenta ACP do czatu.

Jeśli wymagany jest `RAILWISE_API_KEY`, zapoznaj się z sekcją [Konfigurowanie adapterów: Zmienne środowiskowe](https://codecompanion.olimorris.dev/getting-started#setting-an-api-key) w dokumentacji CodeCompanion.nvim, aby uzyskać szczegółowe informacje.

## Wsparcie

RAILWISE (甬算) działa przez ACP tak samo, jak w terminalu. Obsługiwane są wszystkie główne funkcje:

:::note
Niektóre polecenia z ukośnikiem, takie jak `/undo` i `/redo`, nie są obecnie obsługiwane przez ACP.
:::

- Narzędzia wbudowane (operacje na plikach, terminal itp.)
- Niestandardowe narzędzia i polecenia z ukośnikiem
- Serwery MCP zdefiniowane w konfiguracji RAILWISE (甬算)
- Zasady projektu z `AGENTS.md`
- Niestandardowe formatery i lintery
- Agenci systemowi i uprawnienia
