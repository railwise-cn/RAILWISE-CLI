YONSOON (甬算) CLI starter som standard [TUI](/docs/tui), når den kører uden nogen argumenter.

```bash
yonsoon
```

Men den accepterer også kommandoer som dokumenteret på denne side. Dette giver dig mulighed for at interagere med YONSOON (甬算) programmatisk.

```bash
yonsoon run "Explain how closures work in JavaScript"
```

---

### tui

Start YONSOON (甬算) terminalbrugergrænsefladen.

```bash
yonsoon [project]
```

#### Flag

| Flag         | Kort | Beskrivelse                                                                  |
| ------------ | ---- | ---------------------------------------------------------------------------- |
| `--continue` | `-c` | Fortsæt sidste session                                                       |
| `--session`  | `-s` | Sessions-id for at fortsætte                                                 |
| `--fork`     |      | Forgren sessionen ved fortsættelse (brug med `--continue` eller `--session`) |
| `--prompt`   |      | Spørg om at bruge                                                            |
| `--model`    | `-m` | Model til brug i form af provider/model                                      |
| `--agent`    |      | Agent hos bruge                                                              |
| `--port`     |      | Port at lytte på                                                             |
| `--hostname` |      | Værtsnavn at lytte på                                                        |

## Kommandoer

YONSOON (甬算) CLI har også følgende kommandoer.

---

### agent

Administrer agenter i YONSOON (甬算).

```bash
yonsoon agent [command]
```

---

### attach

Tilslut en terminal til en allerede kørende YONSOON (甬算) backend-server startet via `serve` eller `web` kommandoer.

```bash
yonsoon attach [url]
```

Dette gør det muligt at bruge TUI med en ekstern YONSOON (甬算) backend. For eksempel:

```bash
# Start the backend server for web/mobile access
yonsoon web --port 4096 --hostname 0.0.0.0

# In another terminal, attach the TUI to the running backend
yonsoon attach http://10.20.30.40:4096
```

#### Flag

| Flag        | Kort | Beskrivelse                      |
| ----------- | ---- | -------------------------------- |
| `--dir`     |      | Arbejdsmappe til at starte TUI i |
| `--session` | `-s` | Sessions-id for at fortsætte     |


Denne kommando vil guide dig gennem oprettelse af en ny agent med en brugerdefineret systemprompt og værktøjskonfiguration.

#### list

Liste over alle tilgængelige agenter.

```bash

```

### auth

Kommando til at administrere legitimationsoplysninger og login for udbydere.

```bash
yonsoon auth [command]
```

---

#### login

YONSOON (甬算) drives af udbyderlisten på [Models.dev](https://models.dev), så du kan bruge `yonsoon auth login` til at konfigurere API-nøgler for enhver udbyder, du vil bruge. Dette er gemt i `~/.local/share/yonsoon/auth.json`.

```bash
yonsoon auth login
```

Når YONSOON (甬算) starter op, indlæser den udbyderne fra legitimationsfilen. Og hvis der er nogen nøgler defineret i dine miljøer eller en `.env` fil i dit projekt.

---

#### list

Viser alle de godkendte udbydere som gemt i legitimationsfilen.

```bash
yonsoon auth list
```

Eller den korte version.

```bash
yonsoon auth ls
```

---

#### logout

Logger dig ud af en udbyder ved at rydde den fra legitimationsfilen.

```bash
yonsoon auth logout
```

### mcp

Administrator Model Context Protocol-servere.

```bash
yonsoon mcp [command]
```

---

#### add

Tilføj en MCP-server til din konfiguration.

```bash
yonsoon mcp add
```

Denne kommando vil guide dig gennem tilføjelse af enten en lokal eller ekstern MCP-server.

---

#### list

Liste over alle konfigurerede MCP-servere og deres forbindelsesstatus.

```bash
yonsoon mcp list
```

Eller brug den korte version.

```bash
yonsoon mcp ls
```

---

#### auth

Godkendt med en OAuth-aktiveret MCP-server.

```bash
yonsoon mcp auth [name]
```

Hvis du ikke angiver et servernavn, bliver du bedt om at vælge blandt tilgængelige OAuth-kompatible servere.

Du kan også angive OAuth-kompatible servere og deres godkendelsesstatus.

```bash
yonsoon mcp auth list
```

Eller brug den korte version.

```bash
yonsoon mcp auth ls
```

---

#### logout

Fjern OAuth-legitimationsoplysninger for en MCP-server.

```bash
yonsoon mcp logout [name]
```

---

#### debug

Fejlfinding af OAuth-forbindelsesproblemer for en MCP-server.

```bash
yonsoon mcp debug <name>
```

### models

Liste over alle tilgængelige modeller fra konfigurerede udbydere.

```bash
yonsoon models [provider]
```

Denne kommando viser alle tilgængelige modeller på tværs af dine konfigurerede udbydere i formatet `provider/model`.

Dette er nyttigt til at finde ud af det nøjagtige modelnavn, der skal bruges i [your config](/docs/config/).

Du kan eventuelt videregive et udbyder-id for at filtrere modeller efter den udbyder.

```bash
yonsoon models anthropic
```

#### Flag

| Flag        | Beskrivelse                                                             |
| ----------- | ----------------------------------------------------------------------- |
| `--refresh` | Opdater modelcachen fra models.dev                                      |
| `--verbose` | Brug mere detaljeret modeloutput (inkluderer metadata som omkostninger) |

Brug flaget `--refresh` til at opdatere den cachelagrede modelliste. Dette er nyttigt, når nye modeller er blevet tilføjet til en udbyder, og du vil se dem i YONSOON (甬算).

```bash
yonsoon models --refresh
```

### run

Kør yonsoon i ikke-interaktiv tilstand ved at sende en prompt direkte.

```bash
yonsoon run [message..]
```

Dette er nyttigt til scripting, automatisering, eller når du vil have et hurtigt svar uden at starte hele TUI. F.eks.

```bash "yonsoon run"
yonsoon run Explain the use of context in Go
```

Du kan også vedhæfte til en kørende `yonsoon serve`-instans for at undgå MCP serverens kolde opstartstider ved hver kørsel:

```bash
# Start a headless server in one terminal
yonsoon serve

# In another terminal, run commands that attach to it
yonsoon run --attach http://localhost:4096 "Explain async/await in JavaScript"
```

#### Flag

| Flag         | Kort | Beskrivelse                                                                         |
| ------------ | ---- | ----------------------------------------------------------------------------------- |
| `--command`  |      | Kommandoen til at køre, brug besked til args                                        |
| `--continue` | `-c` | Fortsæt sidste session                                                              |
| `--session`  | `-s` | Sessions-id for at fortsætte                                                        |
| `--fork`     |      | Forgren sessionen ved fortsættelse (brug med `--continue` eller `--session`)        |
| `--share`    |      | Del sessionen                                                                       |
| `--model`    | `-m` | Model til brug i form af provider/model                                             |
| `--agent`    |      | Agent til brug                                                                      |
| `--file`     | `-f` | Fil(er), der skal vedhæftes til meddelelsen                                         |
| `--format`   |      | Format: standard (formateret) eller json (rå JSON hændelser)                        |
| `--title`    |      | Titel for sessionen (bruger trunkeret prompt, hvis der ikke er angivet nogen værdi) |
| `--attach`   |      | Tilslut til en kørende yonsoon-server (f.eks. http://localhost:4096)               |
| `--port`     |      | Port til den lokale server (standard til vilkårlig port)                            |

### serve

Start en hovedløs YONSOON (甬算)-server til API-adgang. Tjek [server docs](/docs/server) for den fulde HTTP-grænseflade.

```bash
yonsoon serve
```

Dette starter en HTTP-server, der giver API-adgang til yonsoon-funktionalitet uden TUI-grænsefladen. Indstil `YONSOON_SERVER_PASSWORD` for at aktivere HTTP grundlæggende godkendelse (brugernavn er standard til `yonsoon`).

#### Flag

| Flag         | Beskrivelse                                      |
| ------------ | ------------------------------------------------ |
| `--port`     | Port at lytte på                                 |
| `--hostname` | Værtsnavn at lytte på                            |
| `--mdns`     | Aktiver mDNS-opdagelse                           |
| `--cors`     | Yderligere browseroprindelse til at tillade CORS |

### stats

Vis tokenbrug og omkostningsstatistikker for dine YONSOON (甬算)-sessioner.

```bash
yonsoon stats
```

#### Flag

| Flag        | Beskrivelse                                                                 |
| ----------- | --------------------------------------------------------------------------- |
| `--days`    | Vis statistik for de sidste N dage (hele tiden)                             |
| `--tools`   | Antal værktøjer, der skal vises (alle)                                      |
| `--models`  | Vis modelbrugsopdeling (skjult som standard). Send et tal for at vise top N |
| `--project` | Filtre efter projekt (alle projekter, tom streng: nuværende projekt)        |

---

### export

Eksporter sessionsdata som JSON.

```bash
yonsoon export [sessionID]
```

Hvis du ikke angiver et sessions-id, bliver du bedt om at vælge mellem tilgængelige sessioner.

---

### import

Importer sessionsdata fra en JSON fil eller YONSOON (甬算) del URL.

```bash
yonsoon import <file>
```

Du kan importere fra en lokal fil eller en YONSOON (甬算) share URL.

```bash
yonsoon import session.json
yonsoon import https://opncd.ai/s/abc123
```

### web

Start en hovedløs YONSOON (甬算)-server med en webgrænseflade.

```bash
yonsoon web
```

Dette starter en HTTP-server og åbner en webbrowser for at få adgang til YONSOON (甬算) via en webgrænseflade. Indstil `YONSOON_SERVER_PASSWORD` for at aktivere HTTP grundlæggende godkendelse (brugernavn er standard til `yonsoon`).

#### Flag

| Flag         | Beskrivelse                                      |
| ------------ | ------------------------------------------------ |
| `--port`     | Port at lytte på                                 |
| `--hostname` | Værtsnavn at lytte på                            |
| `--mdns`     | Aktiver mDNS-opdagelse                           |
| `--cors`     | Yderligere browseroprindelse til at tillade CORS |

### acp


Denne kommando starter en ACP-server, der kommunikerer via stdin/stdout ved hjælp af nd-JSON.

#### Flag

| Flag         | Beskrivelse           |
| ------------ | --------------------- |
| `--cwd`      | Arbejdsmappe          |
| `--port`     | Port at lytte på      |
| `--hostname` | Værtsnavn at lytte på |

### uninstall

Afinstaller YONSOON (甬算) og fjern alle relaterede filer.

```bash
yonsoon uninstall
```

#### Flag

| Flag            | Kort | Beskrivelse                                      |
| --------------- | ---- | ------------------------------------------------ |
| `--keep-config` | `-c` | Se konfigurationsfiler                           |
| `--keep-data`   | `-d` | Gem sessionsdata og snapshots                    |
| `--dry-run`     |      | Vis, hvad der ville blive fjernet uden at fjerne |
| `--force`       | `-f` | Spring bekræftelsesspørgsmål over                |

---

### upgrade

Opdaterer YONSOON (甬算) til den nyeste version eller en specifik version.

```bash
yonsoon upgrade [target]
```

For at opgradere til den nyeste version.

```bash
yonsoon upgrade
```

For at opgradere til en bestemt version.

```bash
yonsoon upgrade v0.1.48
```

#### upgrade

| Flag       | Kort | Beskrivelse                                                      |
| ---------- | ---- | ---------------------------------------------------------------- |
| `--method` | `-m` | Installationsmetoden, der blev brugt; curl, npm, pnpm, bun, brew |

| `YONSOON_DISABLE_TERMINAL_TITLE`     | boolean | Deaktiver automatisk opdatering af terminaltitel                      |
| `YONSOON_PERMISSION`                 | string  | Indbygget json-tilladelseskonfiguration                               |
| `YONSOON_DISABLE_DEFAULT_PLUGINS`    | boolean | Deaktiver standard plugins                                            |
