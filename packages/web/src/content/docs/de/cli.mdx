Der RAILWISE (甬算) CLI startet standardmäßig das [TUI](/docs/tui), wenn er ohne Argumente ausgeführt wird.

```bash
railwise
```

Es akzeptiert aber auch Befehle, wie auf dieser Seite dokumentiert. Dadurch können Sie programmgesteuert mit RAILWISE (甬算) interagieren.

```bash
railwise run "Explain how closures work in JavaScript"
```

---

### tui

Starten Sie die Terminalbenutzeroberfläche RAILWISE (甬算).

```bash
railwise [project]
```

#### Optionen

| Flag         | Kurz | Beschreibung                                                           |
| ------------ | ---- | ---------------------------------------------------------------------- |
| `--continue` | `-c` | Setzen Sie die letzte Sitzung fort                                     |
| `--session`  | `-s` | Session-ID zum Fortfahren                                              |
| `--fork`     |      | Sitzung beim Fortsetzen verzweigen (mit `--continue` oder `--session`) |
| `--prompt`   |      | Prompt zur Verwendung                                                  |
| `--model`    | `-m` | Zu verwendendes Modell in der Form provider/model                      |
| `--agent`    |      | Zu verwendender Agent                                                  |
| `--port`     |      | Port zum Abhören                                                       |
| `--hostname` |      | Hostname zum Abhören                                                   |

## Befehle

Der RAILWISE (甬算) CLI verfügt außerdem über die folgenden Befehle.

---

### agent

Agenten für RAILWISE (甬算) verwalten.

```bash
railwise agent [command]
```

---

### attach

Schließen Sie ein Terminal an einen bereits laufenden RAILWISE (甬算)-Backend-Server an, der über die Befehle `serve` oder `web` gestartet wurde.

```bash
railwise attach [url]
```

Dies ermöglicht die Verwendung von TUI mit einem Remote-Backend RAILWISE (甬算). Zum Beispiel:

```bash
# Start the backend server for web/mobile access
railwise web --port 4096 --hostname 0.0.0.0

# In another terminal, attach the TUI to the running backend
railwise attach http://10.20.30.40:4096
```

#### Optionen

| Flag        | Kurz | Beschreibung                              |
| ----------- | ---- | ----------------------------------------- |
| `--dir`     |      | Arbeitsverzeichnis zum Starten von TUI in |
| `--session` | `-s` | Session-ID zum Fortfahren                 |

---

#### create

Erstellen Sie einen neuen Agenten mit benutzerdefinierter Konfiguration.

```bash
railwise agent create
```

Dieser Befehl führt Sie durch die Erstellung eines neuen Agenten mit einer benutzerdefinierten Systemeingabeaufforderung und Toolkonfiguration.

---

#### list

Listen Sie alle verfügbaren Agenten auf.

```bash
railwise agent list
```

### auth

Befehl zum Verwalten von Anmeldeinformationen und Anmelden für Anbieter.

```bash
railwise auth [command]
```

---

#### login

RAILWISE (甬算) wird von der Anbieterliste bei [Models.dev](https://models.dev) unterstützt, sodass Sie `railwise auth login` verwenden können, um API-Schlüssel für jeden Anbieter zu konfigurieren, den Sie verwenden möchten. Dies wird in `~/.local/share/railwise/auth.json` gespeichert.

```bash
railwise auth login
```

Wenn RAILWISE (甬算) startet, lädt es den Anbieter aus der Anmeldeinformationsdatei. Und ob in Ihren Umgebungen Schlüssel sind oder in Ihrem Projekt eine `.env`-Datei vorhanden ist.

---

#### list

Listet alle authentifizierten Anbieter auf, wie in der Anmeldeinformationsdatei gespeichert.

```bash
railwise auth list
```

Oder die Kurzversion.

```bash
railwise auth ls
```

---

#### logout

Melden Sie sich von einem Anbieter ab, indem Sie ihn aus der Anmeldeinformationsdatei löschen.

```bash
railwise auth logout
```

### mcp

Verwalten Sie Model Context Protocol-Server.

```bash
railwise mcp [command]
```

---

#### add

Fügen Sie Ihrer Konfiguration einen MCP-Server hinzu.

```bash
railwise mcp add
```

Dieser Befehl führt Sie durch das Hinzufügen eines lokalen oder Remote-MCP-Servers.

---

#### list

Listen Sie alle konfigurierten MCP-Server und deren Verbindungsstatus auf.

```bash
railwise mcp list
```

Oder nutzen Sie die Kurzversion.

```bash
railwise mcp ls
```

---

#### auth

Authentifizieren Sie sich mit einem OAuth-fähigen MCP-Server.

```bash
railwise mcp auth [name]
```

Wenn Sie keinen Servernamen angeben, werden Sie aufgefordert, einen der verfügbaren OAuth-fähigen Server auszuwählen.

Sie können auch OAuth-fähige Server und deren Authentifizierungsstatus auflisten.

```bash
railwise mcp auth list
```

Oder nutzen Sie die Kurzversion.

```bash
railwise mcp auth ls
```

---

#### logout

Entfernen Sie OAuth-Anmeldeinformationen für einen MCP-Server.

```bash
railwise mcp logout [name]
```

---

#### debug

Beheben Sie OAuth-Verbindungsprobleme für einen MCP-Server.

```bash
railwise mcp debug <name>
```

### models

Listen Sie alle verfügbaren Modelle der konfigurierten Anbieter auf.

```bash
railwise models [provider]
```

Dieser Befehl zeigt alle bei Ihren konfigurierten Anbietern verfügbaren Modelle im Format `provider/model` an.

Dies ist nützlich, um die genauen Modellnamen herauszufinden, die in [Ihrer Konfiguration](/docs/config/) verwendet werden sollen.

Sie können optional eine Anbieter-ID übergeben, um Modelle nach diesem Anbieter zu filtern.

```bash
railwise models anthropic
```

#### Optionen

| Flag        | Beschreibung                                                                          |
| ----------- | ------------------------------------------------------------------------------------- |
| `--refresh` | Aktualisieren Sie den Modellcache von models.dev                                      |
| `--verbose` | Verwenden Sie eine ausführlichere Modellausgabe (einschließlich Metadaten wie Kosten) |

Verwenden Sie das Flag `--refresh`, um die zwischengespeicherte Modellliste zu aktualisieren. Dies ist nützlich, wenn einem Anbieter neue Modelle hinzugefügt wurden und Sie diese in RAILWISE (甬算) sehen möchten.

```bash
railwise models --refresh
```

### run

Führen Sie RAILWISE (甬算) im nicht interaktiven Modus aus, indem Sie direkt eine Eingabeaufforderung übergeben.

```bash
railwise run [message..]
```

Dies ist nützlich für die Skripterstellung, die Automatisierung oder wenn Sie eine schnelle Antwort wünschen, ohne das vollständige TUI zu starten. Zum Beispiel:

```bash "railwise run"
railwise run Explain the use of context in Go
```

Sie können auch eine Verbindung zu einer laufenden `railwise serve`-Instanz herstellen, um bei jedem Lauf Kaltstartzeiten für den MCP-Server zu vermeiden:

```bash
# Start a headless server in one terminal
railwise serve

# In another terminal, run commands that attach to it
railwise run --attach http://localhost:4096 "Explain async/await in JavaScript"
```

#### Optionen

| Flag         | Kurz | Beschreibung                                                                                        |
| ------------ | ---- | --------------------------------------------------------------------------------------------------- |
| `--command`  |      | Der auszuführende Befehl, Argumente als Nachricht verwenden                                         |
| `--continue` | `-c` | Setzen Sie die letzte Sitzung fort                                                                  |
| `--session`  | `-s` | Session-ID zum Fortfahren                                                                           |
| `--fork`     |      | Sitzung beim Fortsetzen verzweigen (mit `--continue` oder `--session` verwenden)                    |
| `--share`    |      | Teilen Sie die Sitzung                                                                              |
| `--model`    | `-m` | Zu verwendendes Modell in der Form provider/model                                                   |
| `--agent`    |      | Zu verwendender Agent                                                                               |
| `--file`     | `-f` | Datei(en) zum Anhängen an die Nachricht                                                             |
| `--format`   |      | Format: default (formatiert) oder json (rohe JSON-Ereignisse)                                       |
| `--title`    |      | Titel für die Sitzung (verwendet eine verkürzte Eingabeaufforderung, wenn kein Wert angegeben wird) |
| `--attach`   |      | An einen laufenden RAILWISE (甬算)-Server anschließen (z.B. http://localhost:4096)                         |
| `--port`     |      | Port für den lokalen Server (standardmäßig zufälliger Port)                                         |

### serve

Starten Sie einen Headless-RAILWISE (甬算)-Server für den API-Zugriff. Sehen Sie sich die [Server-Dokumentation](/docs/server) für die vollständige HTTP-Schnittstelle an.

```bash
railwise serve
```

Dadurch wird ein HTTP-Server gestartet, der API-Zugriff auf RAILWISE (甬算)-Funktionalität ohne die TUI-Schnittstelle bietet. Legen Sie `RAILWISE_SERVER_PASSWORD` fest, um die HTTP-Basisauthentifizierung zu aktivieren (Benutzername ist standardmäßig `railwise`).

#### Optionen

| Flag         | Beschreibung                                    |
| ------------ | ----------------------------------------------- |
| `--port`     | Port zum Abhören                                |
| `--hostname` | Hostname zum Abhören                            |
| `--mdns`     | mDNS-Discovery aktivieren                       |
| `--cors`     | Zusätzliche Browser-Ursprünge für CORS zulassen |

### stats

Zeigen Sie Token-Nutzungs- und Kostenstatistiken für Ihre RAILWISE (甬算)-Sitzungen an.

```bash
railwise stats
```

#### Optionen

| Flag        | Beschreibung                                                                                                                   |
| ----------- | ------------------------------------------------------------------------------------------------------------------------------ |
| `--days`    | Statistiken für die letzten N Tage anzeigen (alle Zeiten)                                                                      |
| `--tools`   | Anzahl der angebotenen Werkzeuge (alle)                                                                                        |
| `--models`  | Aufschlüsselung der Modellnutzung anzeigen (standardmäßig ausgeblendet). Übergeben Sie eine Zahl, um die obersten N anzuzeigen |
| `--project` | Nach Projekt filtern (alle Projekte, leere Zeichenfolge: aktuelles Projekt)                                                    |

### export

Sitzungsdaten als JSON exportieren.

```bash
railwise export [sessionID]
```

Wenn Sie keine Sitzungs-ID angeben, werden Sie aufgefordert, eine der verfügbaren Sitzungen auszuwählen.

---

### import

Importieren Sie Sitzungsdaten aus einer JSON-Datei oder einer RAILWISE (甬算)-Freigabe-URL.

```bash
railwise import <file>
```

Sie können aus einer lokalen Datei oder einer RAILWISE (甬算)-Freigabe-URL importieren.

```bash
railwise import session.json
railwise import https://opncd.ai/s/abc123
```

| ------------ | ----------------------------------------------- |
| `--port`     | Port zum Abhören                                |
| `--hostname` | Hostname zum Abhören                            |
| `--mdns`     | mDNS-Discovery aktivieren                       |
| `--cors`     | Zusätzliche Browser-Ursprünge für CORS zulassen |

#### Optionen

### uninstall

Deinstallieren Sie RAILWISE (甬算) und entfernen Sie alle zugehörigen Dateien.

```bash
railwise uninstall
```

#### Optionen

| Flag            | Kurz | Beschreibung                                        |
| --------------- | ---- | --------------------------------------------------- |
| `--keep-config` | `-c` | Konfigurationsdateien behalten                      |
| `--keep-data`   | `-d` | Sitzungsdaten und Snapshots aufbewahren             |
| `--dry-run`     |      | Zeigt, was entfernt werden würde, ohne zu entfernen |
| `--force`       | `-f` | Bestätigungsaufforderungen überspringen             |

---

### upgrade

Aktualisiert RAILWISE (甬算) auf die neueste Version oder eine bestimmte Version.

```bash
railwise upgrade [target]
```

Um auf die neueste Version zu aktualisieren:

```bash
railwise upgrade
```

Um auf eine bestimmte Version zu aktualisieren.

```bash
railwise upgrade v0.1.48
```

#### Optionen

| Flag       | Kurz | Beschreibung                                                        |
| ---------- | ---- | ------------------------------------------------------------------- |
| `--method` | `-m` | Die zu verwendende Installationsmethode; curl, npm, pnpm, bun, brew |
