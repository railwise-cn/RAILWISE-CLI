Railwise CLI 在不帶任何參數執行時，預設啟動 [TUI](/docs/tui)。

```bash
railwise
```

但它也接受本頁面中記錄的指令，使您可以透過程式方式與 Railwise 進行互動。

```bash
railwise run "Explain how closures work in JavaScript"
```

---

### tui

啟動 Railwise 終端機使用者介面。

```bash
railwise [project]
```

#### 旗標

| 旗標         | 簡寫 | 說明                                                          |
| ------------ | ---- | ------------------------------------------------------------- |
| `--continue` | `-c` | 繼續上一個工作階段                                            |
| `--session`  | `-s` | 要繼續的工作階段 ID                                           |
| `--fork`     |      | 繼續時分岔工作階段（與 `--continue` 或 `--session` 搭配使用） |
| `--prompt`   |      | 要使用的提示詞                                                |
| `--model`    | `-m` | 要使用的模型，格式為 provider/model                           |
| `--agent`    |      | 要使用的代理                                                  |
| `--port`     |      | 監聽連接埠                                                    |
| `--hostname` |      | 監聽主機名稱                                                  |

## 指令

Railwise CLI 還提供以下指令。

---

### agent

管理 Railwise 的代理。

```bash
railwise agent [command]
```

---

### attach

將終端機連接到已透過 `serve` 或 `web` 指令啟動的 Railwise 後端伺服器。

```bash
railwise attach [url]
```

這允許將 TUI 與遠端 Railwise 後端搭配使用。例如：

```bash
# Start the backend server for web/mobile access
railwise web --port 4096 --hostname 0.0.0.0

# In another terminal, attach the TUI to the running backend
railwise attach http://10.20.30.40:4096
```

#### 旗標

| 旗標        | 簡寫 | 說明                |
| ----------- | ---- | ------------------- |
| `--dir`     |      | 啟動 TUI 的工作目錄 |
| `--session` | `-s` | 要繼續的工作階段 ID |

---

#### create

使用自訂設定建立新的代理。

```bash
railwise agent create
```

此指令將引導您使用自訂系統提示詞和工具設定來建立新的代理。

---

#### list

列出所有可用的代理。

```bash
railwise agent list
```

### auth

管理供應商的憑證和登入資訊的指令。

```bash
railwise auth [command]
```

---

#### login

Railwise 基於 [Models.dev](https://models.dev) 的供應商列表運作，因此您可以使用 `railwise auth login` 為任何想要使用的供應商設定 API 金鑰。金鑰儲存在 `~/.local/share/railwise/auth.json` 中。

```bash
railwise auth login
```

Railwise 啟動時會從憑證檔案載入供應商資訊，同時也會載入環境變數或專案中 `.env` 檔案中定義的金鑰。

---

#### list

列出憑證檔案中儲存的所有已認證供應商。

```bash
railwise auth list
```

或使用簡寫版本。

```bash
railwise auth ls
```

---

#### logout

從憑證檔案中清除供應商資訊以完成登出。

```bash
railwise auth logout
```

---

### github

管理用於儲存庫自動化的 GitHub 代理。

```bash
railwise github [command]
```

---

#### install

在您的儲存庫中安裝 GitHub 代理。

```bash
railwise github install
```

此指令會設定必要的 GitHub Actions 工作流程並引導您完成設定過程。[了解更多](/docs/github)。

---

#### run

執行 GitHub 代理。通常在 GitHub Actions 中使用。

```bash
railwise github run
```

##### 旗標

| 旗標      | 說明                           |
| --------- | ------------------------------ |
| `--event` | 用於執行代理的 GitHub 模擬事件 |
| `--token` | GitHub 個人存取權杖            |

### mcp

管理 Model Context Protocol 伺服器。

```bash
railwise mcp [command]
```

---

#### add

將 MCP 伺服器新增到您的設定中。

```bash
railwise mcp add
```

此指令將引導您新增本地或遠端 MCP 伺服器。

---

#### list

列出所有已設定的 MCP 伺服器及其連線狀態。

```bash
railwise mcp list
```

或使用簡寫版本。

```bash
railwise mcp ls
```

---

#### auth

對支援 OAuth 的 MCP 伺服器進行認證。

```bash
railwise mcp auth [name]
```

如果您不提供伺服器名稱，系統將提示您從可用的支援 OAuth 的伺服器中進行選擇。

您還可以列出支援 OAuth 的伺服器及其認證狀態。

```bash
railwise mcp auth list
```

或使用簡寫版本。

```bash
railwise mcp auth ls
```

---

#### logout

移除 MCP 伺服器的 OAuth 憑證。

```bash
railwise mcp logout [name]
```

---

#### debug

偵錯 MCP 伺服器的 OAuth 連線問題。

```bash
railwise mcp debug <name>
```

---

### models

列出已設定供應商的所有可用模型。

```bash
railwise models [provider]
```

此指令以 `provider/model` 的格式顯示所有已設定供應商中可用的模型。

這對於確定在[設定檔](/docs/config/)中使用的確切模型名稱非常有用。

您可以選擇傳入供應商 ID 來按供應商篩選模型。

```bash
railwise models anthropic
```

#### 旗標

| 旗標        | 說明                                       |
| ----------- | ------------------------------------------ |
| `--refresh` | 從 models.dev 重新整理模型快取             |
| `--verbose` | 使用更詳細的模型輸出（包含費用等中繼資料） |

使用 `--refresh` 旗標可以更新快取的模型列表。當供應商新增了模型並且您希望在 Railwise 中看到它們時，此功能非常有用。

```bash
railwise models --refresh
```

### run

以非互動模式執行 Railwise，直接傳入提示詞。

```bash
railwise run [message..]
```

這對於指令碼編寫、自動化或無需啟動完整 TUI 即可快速取得答案的情境非常有用。例如：

```bash "railwise run"
railwise run Explain the use of context in Go
```

您還可以連接到正在執行的 `railwise serve` 實例，以避免每次執行時 MCP 伺服器的冷啟動時間：

```bash
# Start a headless server in one terminal
railwise serve

# In another terminal, run commands that attach to it
railwise run --attach http://localhost:4096 "Explain async/await in JavaScript"
```

#### 旗標

| 旗標         | 簡寫 | 說明                                                           |
| ------------ | ---- | -------------------------------------------------------------- |
| `--command`  |      | 要執行的指令，使用 message 作為參數                            |
| `--continue` | `-c` | 繼續上一個工作階段                                             |
| `--session`  | `-s` | 要繼續的工作階段 ID                                            |
| `--fork`     |      | 繼續時分岔工作階段（與 `--continue` 或 `--session` 搭配使用）  |
| `--share`    |      | 分享工作階段                                                   |
| `--model`    | `-m` | 要使用的模型，格式為 provider/model                            |
| `--agent`    |      | 要使用的代理                                                   |
| `--file`     | `-f` | 附加到訊息的檔案                                               |
| `--format`   |      | 格式：default（格式化輸出）或 json（原始 JSON 事件）           |
| `--title`    |      | 工作階段標題（未提供值時使用截斷的提示詞）                     |
| `--attach`   |      | 連接到正在執行的 railwise 伺服器（例如 http://localhost:4096） |
| `--port`     |      | 本地伺服器連接埠（預設為隨機連接埠）                           |

### serve

啟動無介面的 Railwise 伺服器以提供 API 存取。查看[伺服器文件](/docs/server)了解完整的 HTTP 介面。

```bash
railwise serve
```

此指令啟動一個 HTTP 伺服器，提供對 Railwise 功能的 API 存取，無需 TUI 介面。設定 `RAILWISE_SERVER_PASSWORD` 可啟用 HTTP 基本認證（使用者名稱預設為 `railwise`）。

#### 旗標

| 旗標         | 說明                       |
| ------------ | -------------------------- |
| `--port`     | 監聽連接埠                 |
| `--hostname` | 監聽主機名稱               |
| `--mdns`     | 啟用 mDNS 探索             |
| `--cors`     | 允許 CORS 的額外瀏覽器來源 |

### session

管理 Railwise 工作階段。

```bash
railwise session [command]
```

---

#### list

列出所有 Railwise 工作階段。

```bash
railwise session list
```

##### 旗標

| 旗標          | 簡寫 | 說明                                  |
| ------------- | ---- | ------------------------------------- |
| `--max-count` | `-n` | 限制為最近 N 個工作階段               |
| `--format`    |      | 輸出格式：table 或 json（預設 table） |

---

### stats

顯示 Railwise 工作階段的 Token 用量和費用統計資訊。

```bash
railwise stats
```

#### 旗標

| 旗標        | 說明                                                 |
| ----------- | ---------------------------------------------------- |
| `--days`    | 顯示最近 N 天的統計資訊（預設為所有時間）            |
| `--tools`   | 顯示的工具數量（預設為全部）                         |
| `--models`  | 顯示模型用量明細（預設隱藏）。傳入數字可顯示前 N 個  |
| `--project` | 按專案篩選（預設為所有專案，傳入空字串表示當前專案） |

---

### export

將工作階段資料匯出為 JSON。

```bash
railwise export [sessionID]
```

如果您不提供工作階段 ID，系統將提示您從可用的工作階段中進行選擇。

---

### import

從 JSON 檔案或 Railwise 分享連結匯入工作階段資料。

```bash
railwise import <file>
```

您可以從本地檔案或 Railwise 分享連結匯入。

```bash
railwise import session.json
railwise import https://opncd.ai/s/abc123
```

---

### web

啟動帶有 Web 介面的無介面 Railwise 伺服器。

```bash
railwise web
```

此指令啟動一個 HTTP 伺服器並開啟瀏覽器，透過 Web 介面存取 Railwise。設定 `RAILWISE_SERVER_PASSWORD` 可啟用 HTTP 基本認證（使用者名稱預設為 `railwise`）。

#### 旗標

| 旗標         | 說明                       |
| ------------ | -------------------------- |
| `--port`     | 監聽連接埠                 |
| `--hostname` | 監聽主機名稱               |
| `--mdns`     | 啟用 mDNS 探索             |
| `--cors`     | 允許 CORS 的額外瀏覽器來源 |

---

### acp

啟動 ACP（Agent Client Protocol）伺服器。

```bash
railwise acp
```

此指令啟動一個透過 stdin/stdout 使用 nd-JSON 進行通訊的 ACP 伺服器。

#### 旗標

| 旗標         | 說明         |
| ------------ | ------------ |
| `--cwd`      | 工作目錄     |
| `--port`     | 監聽連接埠   |
| `--hostname` | 監聽主機名稱 |

---

### uninstall

解除安裝 Railwise 並刪除所有相關檔案。

```bash
railwise uninstall
```

#### 旗標

| 旗標            | 簡寫 | 說明                           |
| --------------- | ---- | ------------------------------ |
| `--keep-config` | `-c` | 保留設定檔                     |
| `--keep-data`   | `-d` | 保留工作階段資料和快照         |
| `--dry-run`     |      | 顯示將被刪除的內容但不實際刪除 |
| `--force`       | `-f` | 跳過確認提示                   |

---

### upgrade

將 Railwise 更新到最新版本或指定版本。

```bash
railwise upgrade [target]
```

更新到最新版本。

```bash
railwise upgrade
```

更新到指定版本。

```bash
railwise upgrade v0.1.48
```

#### 旗標

| 旗標       | 簡寫 | 說明                                       |
| ---------- | ---- | ------------------------------------------ |
| `--method` | `-m` | 使用的安裝方式：curl、npm、pnpm、bun、brew |

## 全域旗標

Railwise CLI 接受以下全域旗標。

| 旗標           | 簡寫 | 說明                                 |
| -------------- | ---- | ------------------------------------ |
| `--help`       | `-h` | 顯示說明資訊                         |
| `--version`    | `-v` | 印出版本號                           |
| `--print-logs` |      | 將日誌輸出到 stderr                  |
| `--log-level`  |      | 日誌等級（DEBUG、INFO、WARN、ERROR） |

## 環境變數

Railwise 可以透過環境變數進行設定。

| 變數                                  | 類型    | 說明                                        |
| ------------------------------------- | ------- | ------------------------------------------- |
| `RAILWISE_AUTO_SHARE`                 | boolean | 自動分享工作階段                            |
| `RAILWISE_GIT_BASH_PATH`              | string  | Windows 上 Git Bash 可執行檔的路徑          |
| `RAILWISE_CONFIG`                     | string  | 設定檔路徑                                  |
| `RAILWISE_CONFIG_DIR`                 | string  | 設定目錄路徑                                |
| `RAILWISE_CONFIG_CONTENT`             | string  | 內嵌 JSON 設定內容                          |
| `RAILWISE_DISABLE_AUTOUPDATE`         | boolean | 停用自動更新檢查                            |
| `RAILWISE_DISABLE_PRUNE`              | boolean | 停用舊資料清理                              |
| `RAILWISE_DISABLE_TERMINAL_TITLE`     | boolean | 停用自動終端機標題更新                      |
| `RAILWISE_PERMISSION`                 | string  | 內嵌 JSON 權限設定                          |
| `RAILWISE_DISABLE_DEFAULT_PLUGINS`    | boolean | 停用預設外掛程式                            |
| `RAILWISE_DISABLE_LSP_DOWNLOAD`       | boolean | 停用 LSP 伺服器自動下載                     |
| `RAILWISE_ENABLE_EXPERIMENTAL_MODELS` | boolean | 啟用實驗性模型                              |
| `RAILWISE_DISABLE_AUTOCOMPACT`        | boolean | 停用自動上下文壓縮                          |
| `RAILWISE_DISABLE_CLAUDE_CODE`        | boolean | 停用讀取 `.claude`（提示詞 + 技能）         |
| `RAILWISE_DISABLE_CLAUDE_CODE_PROMPT` | boolean | 停用讀取 `~/.claude/CLAUDE.md`              |
| `RAILWISE_DISABLE_CLAUDE_CODE_SKILLS` | boolean | 停用載入 `.claude/skills`                   |
| `RAILWISE_DISABLE_MODELS_FETCH`       | boolean | 停用從遠端來源擷取模型                      |
| `RAILWISE_FAKE_VCS`                   | string  | 用於測試目的的模擬 VCS 供應商               |
| `RAILWISE_DISABLE_FILETIME_CHECK`     | boolean | 停用檔案時間檢查最佳化                      |
| `RAILWISE_CLIENT`                     | string  | 用戶端識別碼（預設為 `cli`）                |
| `RAILWISE_ENABLE_EXA`                 | boolean | 啟用 Exa 網路搜尋工具                       |
| `RAILWISE_SERVER_PASSWORD`            | string  | 為 `serve`/`web` 啟用基本認證               |
| `RAILWISE_SERVER_USERNAME`            | string  | 覆寫基本認證使用者名稱（預設為 `railwise`） |
| `RAILWISE_MODELS_URL`                 | string  | 自訂模型設定擷取 URL                        |
